## Slide 4 — 핵심 차별화: VALIDATOR

### Python 룰 엔진 37개 — 직접 코드로 구현

| 카테고리 | 룰 수 | 예시 |
| -------- | ----- | ---- |
| 단가 이상치 | 8개 | m² 150,000원 미만 → error |
| 누락 공종 탐지 | 12개 | 욕실 방수 누락 → warning |
| 비율 이상치 | 7개 | 전기 비중 3% 미만 → error |
| 욕실 기준 | 5개 | 욕실 단품 150만원 미만 → error |
| 주방 기준 | 3개 | 주방 가전 없이 싱크대만 → warning |
| 신축 기준 | 2개 | 신축 전기 비중 8% 미만 → error |

**GPT/Gemini와의 차이**:  
AI에게 "전문가처럼 행동해라"고 시키는 게 아니라,  
**18년 현장 경험이 Python 코드 자체에 내재되어 있다.**

## Slide 6 — 데모 결과 (30평 아파트)

### 입력

- 공사 유형: 인테리어
- 면적: 99m²
- 설명: 전체 인테리어, 욕실 2개 교체, 강마루, 실크도배

### 출력 (약 12초)

- 견적 범위: **2,200만원 ~ 3,800만원**
- 기준 단가: 275,000원/m²
- 공종 내역: 14개 공종 자동 분류
- VALIDATOR: 정상 판정 + 전문가 총평 자동 생성 (Python 룰 엔진 37개 전체 통과)
- PDF: 즉시 다운로드

## Slide 7-B — 창의성: 왜 이 접근법이 새로운가

### "룰을 코드로 쓴다" — 업계 최초 현장 룰셋 명시 구현

| 기존 AI 견적 | JH EstimateAI |
| ----------- | ------------- |
| 프롬프트: "전문가처럼 견적해줘" | Python 코드: `if 욕실 and not 방수 → error` |
| 매번 다른 결과 (비결정적) | 동일 입력 → 동일 판정 (결정적) |
| 이상치 탐지 불가 | 37개 룰 기반 이상치 즉시 플래그 |
| 현장 경험 없음 | 18년 시공 현장 데이터 내재화 |

**창의적 핵심**: AI와 도메인 전문성의 결합 방식을 역전시켰다.  
AI가 전문가를 흉내내는 게 아니라, **전문가의 판단 기준이 코드로 고착화**되고 AI는 언어 처리에 집중한다.
